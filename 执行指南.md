# 执行指南 — AI 如何使用本 Skill

> 本文档指导 AI 如何按序读取和执行 build-agent-system 的 8 个 Phase。
> **先读这篇，再读目录。**

## 执行流程

```
第 1 步：读 build-agent-system.md       → 了解全貌
第 2 步：按 Phase 顺序执行              → Phase 1 → 2 → 3 → ...
第 3 步：进入一个 Phase 先读 index.md   → 了解该 Phase 有几个步骤
第 4 步：逐个执行 step 文件             → 每步只读对应的 step-*.md
第 5 步：全部步骤完成后读 verification.md → 逐条验证
第 6 步：验证通过 → 进入下一个 Phase
第 7 步：验证不通过 → 修复后再验证
```

## 核心规则

1. **按序执行：** 每个 Phase 必须按编号顺序执行，前一个验证通过后才进入下一个
2. **每步必验：** 每个 Phase 完成后必须跑 verification.md 中的每条验证
3. **MVP 优先：** 先走 MVP 路径（Phase 1 → 2 → 4）跑通"能对话、能调工具"的最小 Agent，再回头补其他 Phase
4. **按需深读：** 执行某一步时只读对应的 step-*.md，不需要一次加载整个 Phase 的内容
5. **输出产物：** 每个 Phase 标明了输出产物（如 `loop_engine.go`、`tool_registry.go`），执行完应产出对应文件

## Phase 依赖关系

```
Phase 1 (Loop Engineering)     ← 无前置，最先做
    ↓
Phase 2 (Core Architecture)    ← 需要 Phase 1 的循环范式
    ↓
Phase 3 (Memory System)        ← 需要 Phase 2 的 Session 管理
    ↓
Phase 4 (Tool System)          ← 需要 Phase 2 的 Tool Registry
    ↓
Phase 5 (RAG Pipeline)         ← 需要 Phase 2 + Phase 4
    ↓
Phase 6 (Model Optimization)   ← 可并行，贯穿全过程
    ↓
Phase 7 (Multi-Agent)          ← 需要 Phase 2 + Phase 4
    ↓
Phase 8 (Harness Engineering)  ← 需要 Phase 1-7 全部完成
```

## MVP 路径（最小可行产品）

如果只想快速跑通一个最简 Agent：

```
Phase 1 → 选 ReAct 范式（最快路径）
Phase 2 → 搭骨架 + Tool Registry
Phase 4 → 只做 FC，不接 MCP
结果：得到一个"能对话、能调工具"的最简 Agent
然后：Phase 3 → Phase 5 → Phase 6 → Phase 7 → Phase 8
```

## 常见问题

**AI 应该先读哪个文件？**
→ 先读本文件（执行指南.md），再读 build-agent-system.md。

**每一步都要验证吗？**
→ 不是每步单独验证，是整个 Phase 的步骤全部完成后，一起跑 verification.md。

**验证没通过怎么办？**
→ 排查失败原因，修改对应代码，重新验证，通过后才进入下一个 Phase。

**MVP 路径和其他 Phase 的关系？**
→ MVP 走完 Phase 1→2→4 后得到一个可用的最小 Agent。之后按需补 Phase 3、5、6、7、8。
