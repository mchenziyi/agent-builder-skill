# Step 4.6 — 设计安全边界

> 敏感工具权限确认 + 参数校验 + 沙箱。

## 风险与防护

| 风险场景 | 防护措施 |
|---|---|
| 用户诱导执行危险工具 | 敏感工具加权限确认（「确认要删除吗？」）|
| 参数注入（SQL 注入等） | 参数校验 + 参数化查询 |
| 文件系统越权 | Agent 限定沙箱目录 |
| 高频 API 调用消耗费用 | 速率限制 + 会话预算上限 |
| 工具返回恶意内容 | 结果长度限制 + 内容过滤 |

## 前置条件

- Tool Registry 已实现（step-2-4）
- 至少有一个"危险"工具（如文件删除、数据修改）已注册

## AI 必须产出

- 安全校验模块（权限确认 + 参数校验 + 沙箱）
- 文件名按目标项目语言和目录结构决定

## 最小接口契约

- **输入**：工具调用请求（name + args）
- **输出**：允许执行 / 拒绝执行 / 需用户确认
- **不变量**：任何被标记为敏感的工具，必须经过确认才能执行

## 完成定义

- 敏感工具被调用时触发权限确认
- 用户拒绝后工具不被执行
- 对应 verification.md V5 测试通过

## FAQ

**Q: 参数校验该在哪一层做？**
A: 两层都要。Tool Registry 入口做基本类型校验，Handler 内部做业务校验。
