## V3 — 压缩效果

| 步骤 | 操作 | 预期结果 |
|---|---|---|
| 1 | 发起 15 轮对话 | — |
| 2 | 触发压缩 | 成功压缩 |
| 3 | 检查压缩前后的 Token 数 | 压缩后 < 压缩前 50% |
| 4 | 检查压缩后关键信息是否保留 | 关键事实（用户名字、偏好）未丢失 |

- [ ] Token 消耗显著下降
## 验证协议

### V1 — 短期记忆滑动窗口
1. 设置短期记忆 `MAX_MESSAGES=3`（或按 Token 预算）
2. 依次追加 4 条消息，断言消息数不超过 3
3. 断言最早的消息（第 1 条）已被丢弃

### V2 — 长期记忆检索
1. 存储一条记忆：内容为"用户喜欢蓝色"
2. 新建会话，不注入该记忆
3. 发起检索查询："我喜欢什么颜色"
4. 断言：检索结果中包含"蓝色"

### V3 — 压缩效果
1. 模拟 15 轮对话
2. 触发压缩，记录压缩前后的 Token 数
3. 断言：压缩后 Token 数 < 压缩前 50%
4. 断言：压缩后关键信息（如用户名字、重要决策）未丢失

> 用 `go test` 运行，通过即视为 Phase 3 通过。
