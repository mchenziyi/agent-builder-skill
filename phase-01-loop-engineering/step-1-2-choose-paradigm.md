# Step 1.2 — 根据场景选择主范式

| 场景 | 推荐 | 理由 |
|---|---|---|
| 高实时、多轮对话 | ReAct | 延迟低，快速响应 |
| 多步骤、依赖链长 | Plan-and-Execute | 避免短视决策 |
| 高准确率、可纠错 | Reflection | 出错能自我修正 |
| 混合场景 | ReAct + Plan-and-Execute | ReAct 兜底，复杂子任务用 Plan |
| 生产级系统 | ReAct + Reflection | ReAct 主循环 + 关键步骤反思 |

## 前置条件

- Phase 1 前序步骤已完成（理解三种范式）
- 目标项目已确定编程语言和技术栈

## AI 必须产出

- 一个明确的范式选择决策记录（可写入项目文档或注释）
- 文件名按目标项目语言和目录结构决定

## 最小接口契约

- **输入**：业务场景需求文档或面试题上下文
- **输出**：选定的主循环范式（ReAct / Plan-and-Execute / Reflection 或组合）
- **不变量**：选择理由必须记录，后续可追溯

## 完成定义

- 选定了至少一种主循环范式，理由清晰
- 结论已记录到项目文档中
- 对应 verification.md 中 V1 的准备步骤
