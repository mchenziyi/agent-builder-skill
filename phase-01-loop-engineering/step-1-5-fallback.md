# Step 1.5 — 实现 Fallback 策略

| 失败场景 | Fallback |
|---|---|
| 超时 | "处理超时，请简化后重试" |
| 连续 3 次工具失败 | 降级到纯文本回复模式 |
| 循环检测触发 | "检测到重复循环，已终止" |
| API 错误 | 重试 3 次 → 切到备用模型 |

## 前置条件

- 终止条件已实现（step-1-4）
- 已有 Provider 基础实现（至少一个模型可调用）

## AI 必须产出

- Fallback 策略模块
- 文件名按目标项目语言和目录结构决定

## 最小接口契约

- **输入**：失败类型（timeout / tool_error / loop_detected / api_error）
- **输出**：用户可见的友好消息 + 内部状态切换（如降级标记）
- **不变量**：任何失败场景下 Agent 不 panic，始终返回字符串

## 完成定义

- 每种失败场景的 fallback 行为均可独立验证
- 对应 verification.md V3 测试通过
