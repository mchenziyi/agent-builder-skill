## 验证协议

### V1 — Prompt Caching 效果
1. 记录 baseline：发送一次较短请求（仅 system prompt + 一条消息），记录响应时间和 token 数
2. 在短时间内发送相同 system prompt 但不同消息的第二次请求
3. 若 provider 支持自动缓存，第二次请求的响应时间应显著降低（如 < 80%）
4. 若 provider 不支持自动缓存，此项可通过文档确认

### V2 — 解码参数调优
1. 用工具调用参数（temperature=0.1）调工具 10 次，记录每次是否调用了正确工具
2. 用创意参数（temperature=0.8）生成文案 10 次，记录每次输出的多样性
3. 断言：工具调用组正确率 > 90%，创意组输出有可见差异

### V3 — 选型评估
1. 列出你的核心业务场景（至少 3 个），分配权重（总和 100%）
2. 对候选模型逐项打分（1-10）
3. 加权计算总分，排序
4. 用真实业务任务对 top-2 模型做实测验证
5. 断言：实测结果与评分表排序一致
